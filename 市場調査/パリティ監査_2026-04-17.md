### 🔴 gaming-pc の 591 点欠損について (最重要)

現行では Shopify の自動コレクションで 607 台の SKU が並ぶが、新サイトでは 16 件しか表示されていない。原因の仮説:
1. 新サイトの `/collections/gaming-pc` は現行と handle が同じだが、新 Shopify テンプレートで商品タグ/コンディションの絞り込みが異なる可能性

カットオーバー前に **必ず原因特定と修正** が必要。

## (f) 重大差分サマリー (🔴 のみ)

- **🔴 [重大] `/collections/gaming-pc` の商品数が 607 → 16 (-97.4%)**。
  **対処**: Hydrogen の collection loader で全商品ページネーションが効いているか確認。タグフィルタや `first: N` の上限が 16 に制限されていないか、Storefront API クエリを再検証する。カットオーバー前の最優先修正項目。

- **🔴 [重大] `/collections/gadgets` の商品数が 223 → 170 (-23.8%)**。
  **対処**: 53 点の欠損 SKU を突合し、(1) 意図した廃番整理なのか (2) データ未投入なのかを判定。データ未投入なら商品同期ジョブを再実行。

- **🔴 [重大] `/pages/yojimaru-pc` (BTO 注文に関して) が新サイトに存在しない**。
  **対処**: 旧ページ内容を新サイトの `/guides` または専用ページに移植し、301 リダイレクトを設定。

- **🔴 [重大] 運営会社へのリンク (親会社 `mining-base.co.jp`) が新サイトフッターから欠落**。
  **対処**: 特定商取引法記載や企業信頼性の観点から必須。新サイトフッター (`/legal/tokushoho` セクション付近) に会社情報リンクを追加。

- **🔴 [重大 (動作不良)] 販売停止 3 IP (millionlive / millionpro / black-desert) が新サイトで 500 エラー**。
  **対処**: 未存在コレクション handle に対しては 404 を返すよう React Router の catch-all / ErrorBoundary を修正。500 は SEO/UX/監視いずれの観点でも NG。

- **⚠️ [要注意] リラックマコレクションが `rirakkuma` → `goods-rilakkuma` に handle 変更**。
  **対処**: 旧 URL から新 URL への 301 リダイレクトをサーバー側に設定。

- **⚠️ [要注意] ブログ記事が 11 件 → 4 件 (見える範囲)**。
  **対処**: Load More 拡張後の実記事数を確認し、7 件の未掲載記事がある場合は移行。

## 調査手法・データソース

- HTML 取得: `curl -sS -L -A "Mozilla/5.0" --max-time 45`
- 商品数カウント: `<a href="/products/<handle>">` を handle 単位で重複排除
- ページネーション: `?page=1..10` で探索し、新規追加 0 件となったページで打ち切り
- 現行サイトは Cloudflare のレート制限 (429) が頻発したため、一部項目は `WebFetch` (Anthropic 側プロキシ) から取得し補完
- UI 表示の "N 個の商品" ラベルがあればそれを権威値として採用
