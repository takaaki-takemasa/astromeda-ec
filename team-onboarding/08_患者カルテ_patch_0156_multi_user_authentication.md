# patch 0156 multi-user 認証 導入手順 (CEO 一回だけ実行)

既存の single-password 共有から、個別ユーザーごとのログインに切り替わりました。
CEO が最初の 1 回だけ行う切り替え手順です。

---

## 所要時間

3 分。

---

## 切り替え前の状態

- 管理画面ログインは ADMIN_PASSWORD 環境変数 1 つでした (bootstrap モード)
- パスワードは Shopify 環境変数 (Oxygen) に保存

## 切り替え後の状態

- 個別ユーザーを作成し、username + password でログイン
- 役割 (owner / admin / editor / viewer) を個別付与
- 誰がいつ何を変更したか AuditLog で追跡可能
- 退職時はそのユーザーだけ無効化できる

---

## 手順

### Step 1: 現行パスワードで管理画面にログイン

ブラウザで:
```
https://astromeda-ec-273085cdf98d80a57b73.o2.myshopify.dev/admin/login
```

既存の ADMIN_PASSWORD を入力 → ログイン。ログイン画面に「🔧 初期セットアップモード」と表示されていれば正常。

### Step 2: 👥 メンバー タブを開く

左サイドバー → **上級者設定** → **メンバー** タブ。

以下のようなセットアップバナーが出ます:
```
🔧 初期セットアップが必要です
メンバー保管用の領域がまだ作られていません。
下のボタンを押すと一度だけ準備が走ります (やり直し OK・既に準備済みなら何もしません)。
[ ✓ 初期化する ]
```

### Step 3: 「初期化する」をクリック

2-3 秒待つと Toast「初期化完了」。以後、メンバー保管領域 (Shopify Metaobject `astromeda_admin_user`) に個別ユーザーを保存できるようになります。

### Step 4: 最初のオーナー (自分) を作る

「⚠ 個別ユーザーが登録されていません」バナー内の「+ 最初のオーナーを作る」ボタンをクリック。

モーダルで:
- **ユーザー名**: `takaaki` (任意・ログイン ID) — 以降のログインはこれを入力
- **表示名**: `武正貴昭` (admin 画面に出る名前)
- **パスワード**: 8 文字以上の好きなパスワード (パスワード管理ツールで生成推奨)
- **役割**: オーナー (強制・変更不可)

「作成する」→ Toast「作成しました」。

### Step 5: ログアウト → 新しいユーザー名でログイン

右上のメニュー → ログアウト (or `/admin/login` に直接移動)。

今度は **ユーザー名欄が表示されている** (bootstrap モードが自動で終わった証拠)。
- ユーザー名: `takaaki` (Step 4 で決めたもの)
- パスワード: Step 4 で決めたもの

→ ログイン成功すれば切り替え完了 ✅

### Step 6: メンバー追加のテスト (任意)

👥 メンバー タブ → 「+ 新規メンバーを追加」:
- ユーザー名: `test-editor`
- 表示名: `テストエディター`
- パスワード: 任意
- 役割: **編集者** (Editor — 商品・コンテンツ編集のみ可。メンバー管理 NG)

作成後、ログアウトして test-editor でログイン → 👥 メンバータブが見えなくなれば RBAC 正常動作 ✅。
確認できたら owner に戻って test-editor を削除。

## Q&A

### Q: 既存の ADMIN_PASSWORD はどうなる?

A: admin_user が 1 件以上作られた時点で **bootstrap モードが自動で無効化** されます。
以後、ADMIN_PASSWORD では login 不可。個別ユーザー経由のみ。

ADMIN_PASSWORD 環境変数自体は残しておいて OK (将来 bootstrap モードに戻したい時の非常用)。

### Q: オーナーを 1 人だけにすると危険では?

A: 複数のオーナーを作る or admin 役を最低 2 人作ることを推奨します。
システム側で「最後のオーナーは削除できない」「自分自身を無効化できない」を強制していますが、
事故防止のため運用でもオーナー 2 名体制を推奨します。

### Q: パスワードを忘れたら?

A:
- **メンバーが忘れた**: オーナー/admin が 👥 メンバータブ → 該当ユーザー行の「🔑 PW」ボタンで再設定
- **オーナー(自分)が忘れた**: Shopify admin にログインできる人であれば、
  `ADMIN_PASSWORD` 環境変数を一時的に bootstrap モードとして使える。
  Oxygen 環境変数 `ADMIN_USER_METAOBJECT_TYPE` の値を手動編集するのは禁止 (データ破損のため)。

### Q: patch 0155 の 04_admin_password_運用ルール.md は無効?

A: Step 5 完了後、04 の「single-password 共有ルール」は過去のものになります。
退職時の手順だけ一部残し: ADMIN_PASSWORD 変更は不要になり、代わりに退職者を 👥 メンバーで無効化する運用に変わります。

## トラブルシュート

### 初期化ボタンを押しても何も起きない

F12 → Network タブ → 「members」で絞り込み → POST が 200 かどうか確認。
500 が返っていれば patch 番号を CEO に問い合わせ。

### ログインはできたがメンバータブが見えない

ログインしたユーザーの役割が viewer/editor だと 👥 メンバーは見えません。owner または admin でログインし直してください。

### 「最後の owner は削除できません」エラーが出る

システム的に正常な動作です。先に **別の owner を 1 人作る** か、役割を「オーナー」以外に変更してから削除してください。
