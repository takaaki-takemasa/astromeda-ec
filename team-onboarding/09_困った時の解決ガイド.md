# 09. 困った時の解決ガイド — 症状から逆引き

「何か変だぞ」と感じたら、症状でこのページを引いてください。
**結論を先に書いて、その下に「なぜ」「次にどうする」を書きます**。

---

## 🆘 緊急度 高 — お客様に影響している

### サイトが真っ白・404・500 エラー

**結論**: CEO はすぐに Cowork で言ってください。「サイトが真っ白です」とだけ。

**具体的にどうする**:
1. URL を確認 (https://shop.mining-base.co.jp が真っ白か)
2. Cowork に画面のスクリーンショットを 1 枚貼り付け
3. 「これを直してください」と一言

**なぜ**: 真っ白は重大。CEO 自身では原因特定不能。Cowork が `git revert` で 4-5 分前のバージョンに戻します。

### 商品の値段が変・在庫数が違う

**結論**: 管理画面で自分で直してください。Cowork は不要。

**具体的にどうする**: お店の運営 → 商品 → 該当商品 → 値段欄を直す → 保存

**なぜ**: 値段・在庫は管理画面 (admin) でリアルタイム反映。コード変更は不要 (`05_コード変更vs管理画面で完結する変更.md` 参照)。

---

## ⚠ 緊急度 中 — 自分の作業が進まない

### 管理画面にログインできない

**症状別の対処**:

| 症状 | 原因 | 対処 |
|---|---|---|
| 「パスワードが正しくありません」 | パスワードを忘れた・間違えた | オーナーに「パスワードを再設定して」と LINE |
| 「ユーザー名が正しくありません」 | ユーザー名を打ち間違えた・退職済 | LINE で確認 |
| 「このユーザーは無効化されています」 | オーナーが意図的に無効化した | オーナーに「再有効化お願いします」と LINE |
| ログイン画面が真っ白 | システム障害 | 5 分待つ → 直らなければ Cowork |

### Cowork (チャット) が反応しない

**結論**: いったん Claude Desktop アプリを再起動してください。

**具体的に**:
1. Mac: 左上のリンゴマーク → Claude を強制終了
2. Windows: タスクバーで右クリック → Claude を閉じる
3. アプリを再起動 → 同じ会話の続きから書き直す

**それでもダメな時**: アプリの問題なのでアプリの再インストール。詳細は `01_メンバー向け_セットアップ手順.md` の Step 3 (アプリ取得手順) を参照。

### 「変更したのに本物のサイトに反映されない」

**結論**: 4-5 分待ってからブラウザで Ctrl+F5 (強制リロード)。

**具体的に**:
1. Cowork に何かを依頼した時刻をメモ
2. 5 分以上経ってもサイトが古いなら Ctrl+F5 (Mac は Cmd+Shift+R)
3. それでも古ければ、Cowork に「[時刻] に依頼した [変更内容] が反映されていません」と確認

**なぜ**: GitHub Actions が裏で 4-5 分かけてデプロイ。それが終わると本物に反映。たまにブラウザがキャッシュを掴んでいる。

### 「これは admin で直すべき? Cowork に頼むべき?」

**結論**: `05_コード変更vs管理画面で完結する変更.md` を見る。それでも分からなければ Cowork に「これは admin で済みますか?」と聞く。

### 「Cowork にどう話しかけたらいいか分からない」

**結論**: `03_やりたいこと集.md` のお手本をコピペして、語尾だけ変える。

**例**:

**結論**: 管理画面 → 困ったとき → 監査ログ で誰がいつ何を変更したか出ます (patch 0156 以降)。

## 🚨 やっちゃいけないこと事故時の対処

### 商品を間違って消した

**手順**:
1. 商品の名前と handle (URL の最後の部分) をメモ
2. 管理画面 → ファイル管理 → ゴミ箱 (もしあれば) で復元
3. 復元できなければ Cowork に「[商品名] (handle: xxx) を間違って消しました。復元できますか?」

**注意**: 復元できないこともあります。`02_絶対にやらないこと.md` の「お金が動く場所」ルールを再読してください。

### コレクションを間違って消した

商品と同様: Cowork に依頼。コレクション内の商品リスト (タグや手動追加) は復元しないことが多いので、Shopify 上で手動再構築が必要になる場合あり。

### パスワードを LINE グループにうっかり投稿した

**緊急度 P0**:
1. すぐにオーナーに LINE 個別チャットで連絡
2. オーナーは管理画面 → 👥 メンバー → 該当ユーザー → 🔑 PW で **新しいパスワードを設定**
3. 投稿を削除 (LINE はログを完全消去できないので念のため履歴を控える)

### GitHub に間違って秘密のものを push した

例: トークン・パスワードをコードに書いてしまった

**手順**:
1. すぐに Cowork に「[ファイル名] に秘密の [何] を間違って push しました」
2. 該当の秘密 (トークン等) を即座に **GitHub・Shopify 側で無効化** (Cowork が手順案内)
3. ログから削除 (Cowork が `git filter-repo` 等で対応)

**07_GitHub_settings_推奨設定.md** の Secret scanning + Push protection を有効化していれば、push 時点でブロックされる確率が上がります (推奨)。

## 🔍 「自分のせいで何かを壊したかも」と感じたら

### 鉄則: **何もしない・触らない・隠さない**

1. 触ったページを閉じる
2. 何をしたか時系列でメモ (「13:05 商品 X を編集 → 13:06 保存ボタン → 13:07 サイトが変」)
3. オーナー or Cowork に「壊した気がします、これしました [メモを貼る]」

**自分で直そうとしないでください**。3 重ガード (admin の確認画面・git の履歴・最後のオーナー保護) のおかげで、原状復帰はだいたいできます。隠して悪化させる方がリスクです。
