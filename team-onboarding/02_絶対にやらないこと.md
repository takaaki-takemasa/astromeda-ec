# 絶対にやらないこと (v2)

Astromeda チーム全員必読。
**話しかけるだけ体制** でも、この 3 つだけは守ってください。

## 鉄則 3 つ

### ① AI に「全部やり直して」「全部消して」は言わない

Cowork は優秀ですが、「全部」という言葉を広く解釈してしまう危険があります。
**具体的に**言う癖をつけてください:

```
× 「全部やり直して」
○ 「さっきの呪術廻戦の説明だけ元に戻して」

× 「全部消して」
○ 「test というタグが付いている商品を確認して、本当に不要なら外してください」
```

### ② AI が「本当にやっていいですか?」と聞いてきたら、まず手を止める

「これを実行すると履歴が消えます」「元に戻せません」等の警告が出たら:

```
「いったん止めてください。CEO に確認します」
```

と返す。そのまま「はい」と言わない。**CEO に LINE で画面写真** を送る。

### ③ 壊した気がしたら、触らない・CEO に連絡

「サイトが変になった気がする」と感じたら、それ以上の操作をせず:

1. **手を止める**
2. **何をしたかメモ** (例: 「IP コラボの呪術廻戦の説明を直して反映したら、トップが真っ白になった」)
3. **CEO に LINE で画面スクショ + 上のメモ**

CEO が 4-5 分で元に戻せます。**自分で直そうとすると二次被害** になります。

## 普段意識してほしいこと

### ⚠ 本番ストア (shop.mining-base.co.jp) で買い物テストしない

カートに入れて決済画面まで行くと、実際の在庫が一時確保されます。
「動作確認したい」ときは **Cowork に「preview サイトで確認したい」と言う** と、CEO 用の preview URL で確認してくれます。

### ⚠ 同じ箇所を 2 人で同時に触らない

例: 呪術廻戦の説明文を A さんが直している時に B さんも直すと、後から直した方が勝つ形になります。

**対策**: 作業前に Cowork に言う:
```
呪術廻戦の説明を直したいです。誰かが触っていないか確認してください
```

Cowork が GitHub の最新状態を確認して教えてくれます。

### ⚠ お金が動く場所はダメ

以下は **絶対に編集しない** (Cowork に依頼もしない):

- 商品の**価格**を本番で変更する (Cowork に言うと admin 経由で変わってしまう)
- 決済プラグインの設定
- 「ディスカウント」「クーポン」タブの新規作成 (テストでも)

これらは **CEO 専任**。必要なら CEO に依頼してください。

### ⚠ お客様の個人情報は扱わない

- 注文情報 / 顧客名 / 住所 / メールアドレスを Cowork に送らない
- これらを扱う必要がある場合は CEO に依頼

## Cowork に聞くときの禁句集

以下のフレーズは **言わない**:

| 禁句 | 理由 | 代わりに |
|---|---|---|
| 「全部消して」 | 範囲が曖昧 | 「〇〇だけ消して」 |
| 「元に戻して」 (時刻不明) | どこまで戻すか不明 | 「さっき直した〇〇を戻して」 |
| 「force で push して」 | 他人の作業が消える | 普通に「反映して」 |
| 「DB をリセットして」 | 復旧不可能 | 代わりを CEO と相談 |
| 「yes と答えて、全部承認して」 | 危険操作の歯止めが消える | 1 つずつ確認して進めてもらう |

## 何も壊さずに練習したい場合

```
「練習モードで、何も保存せずに〇〇を試してみたいです」
```

Cowork が dry-run (実際には反映しない) で見せてくれます。

## 困ったら 3 秒で判断

- 「自分が壊したかも」→ **触らずに CEO**
- 「AI の言う通りにしていいか不安」→ **まず止めて CEO**
- 「用語が分からない」→ **Cowork に聞く (これは壊れない)**

**CEO に聞くのは恥ずかしくない。壊してから聞く方が恥ずかしい。**
