# 管理画面パスワードの運用ルール (v1)

このドキュメントは **Astromeda 管理画面 (admin) のパスワード運用** に関するルールです。

---

## 現状 (2026-04-24)

### 認証モデル: **single-password 共有**

- 管理画面 `https://astromeda-ec-273085cdf98d80a57b73.o2.myshopify.dev/admin` のログインは **1 つのパスワードを全員で共有** しています
- メンバー個別アカウント機能は **未実装** (将来の patch 0156 で対応予定)
- AuditLog (誰が何を変更したか記録) は role 単位 (admin/editor) では残るが、**個人単位では追跡不能**

---

## CEO がやること

### 1. 新メンバーへの共有方法

パスワードは **以下の経路でのみ** メンバーに渡してください:

| OK | NG |
|---|---|
| LINE 個別チャット | グループチャット (Slack/LINE 全員宛) |
| 暗号化メール (S/MIME) | 平文メール |
| 1Password / Bitwarden 共有 | スプレッドシート / Notion 平文 |
| 直接口頭 | git/GitHub repository 内に書く |

### 2. メンバー追加・退職時の手順

**退職者・契約終了が出たら、48 時間以内にパスワード変更**:

1. Shopify Hydrogen 設定 UI でパスワード変更
   ```
   https://admin.shopify.com/store/production-mining-base/hydrogen/1000122846/settings/environments
   ```
   Environment variables → `ADMIN_PASSWORD` を編集
2. Production / Preview 両方の environment で変更
3. 残メンバー全員に新パスワードを再配布

### 3. 監査ログの確認

「誰かが変な変更をした」と感じたら:
- admin タブ → 困ったとき → 監査ログ
- ただし現状は role 単位 (admin/editor 等) でしか分からない
- どの「人」かは、変更時刻 と 当日の作業履歴 (Slack/LINE) を照合して推定

## メンバーがやること

### 1. パスワードの保管

- ブラウザのパスワードマネージャに保存して OK
- **付箋・メモ帳ファイル・チャット履歴・git 内** に書かない
- 自分の Cowork に「次回も使うので覚えておいて」と頼まない (Cowork の記憶は安全だが、第三者にコピーされた時のリスクが残る)

### 2. ログインしたあと

- 30 分以上席を離れる時は **必ずログアウト** (右上の OWNER メニュー → ログアウト)
- 共有 PC ・公衆 Wi-Fi では admin にログインしない

### 3. パスワードを忘れた・分からない

CEO に LINE で聞いてください。決して「Cowork に教えて」と頼まないでください (Cowork は知りません)。

## 「誰がやったか分からない」問題への当面の対策

multi-user 認証が実装されるまでの **暫定運用ルール**:

### A. 重大変更を伴う作業は事前 Slack/LINE 宣言

```
「これから 13:00〜13:30 で『鬼滅の刃コレクション』の商品タグを一括変更します」
```

→ 後から「誰がやった?」となった時に追跡しやすい。

### B. 同時刻に 2 人で同じ箇所を触らない

- 03_やりたいこと集.md の「⚠ 同じ箇所を 2 人で同時に触らない」も合わせて参照

### C. 「壊した気がする」は即 CEO

- 02_絶対にやらないこと.md の「鉄則 ③」参照
- 自分で直そうとせず、CEO が rollback する
