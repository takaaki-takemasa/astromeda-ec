# やりたいこと集 — Cowork への話しかけ方サンプル (v2)

このマニュアルは **Cowork に何と言えば作業が進むか** のサンプル集です。
**そのままコピペして、自分の値だけ書き換えて使ってください**。

---

## 🟢 5 分で出来ること (まずはここから)

### IPコラボの説明文を直したい

```
IP コラボの「呪術廻戦」の説明文を「(新しい説明文)」に変えて反映してください
```

→ Cowork が自動で:
- 関連ファイルを探す
- 中身を直す
- 反映確認まで行う

### トップのヒーローバナーの文言を変えたい

```
トップページのヒーローバナーで「(現在の文言)」となっているところを
「(新しい文言)」に変えて反映してください
```

### ヒーローバナーの画像を差し替えたい

```
トップページのヒーローバナーの画像を差し替えたいです。
新しい画像はここにあります: (URL またはファイルパス)
案内してください
```

### FAQ を 1 問追加したい

```
FAQ に新しい質問を追加してください:
質問: 「(質問文)」
回答: 「(回答文)」
```

### Marquee (流れる文字) を追加したい

```
トップページのマーキー (流れる文字) に「(追加文言)」を追加してください
```

## 🟡 10-30 分で出来ること

### タイトルに特定のキーワードを含む商品全部にタグを付けたい

```
タイトルに「(キーワード)」を含む商品を全部探して、
「(タグ名)」というタグを付けてください
追加前後の件数を教えてください
```

### 新しい IP コラボを追加したい

```
新しい IP コラボ「(IP 名)」を追加したいです。
Shopify には「(コレクション handle)」というコレクションを既に作っています。
追加に必要な情報を教えてください
```

→ Cowork が必要情報 (説明文・カテゴリ・色等) を順番に質問してくれる

### キャンペーンバナーを追加したい

```
新しいキャンペーン「(タイトル)」を (開始日)〜(終了日) で追加してください。
表示位置はトップページです
```

### 商品の説明文を直したい

```
商品「(商品名)」の説明文を直したいです。案内してください
```

→ Cowork が admin のどこを直すか教えてくれる

```

```

### 自分の最近の作業を確認したい

```

```

### 本番サイトの今を確認したい

```
本番サイトの (どのページ) のスクリーンショットを撮って見せてください
変なところが無いか教えてください
```

## 🚨 緊急対応

### 「サイトが見えない」と通報があった

```
本番サイトを開いて、見えるかどうか確認してください。
HTTP エラーやコンソールエラーがあれば教えてください。
直さずに切り分けまで報告してください
```

→ Cowork が自分で切り分けて報告。修復は CEO 判断後

### 自分が壊した気がする

```
さっき (何分前 / どの作業) を行った後から、
(どこ) の表示がおかしくなっています。
止めてください。CEO に連絡したいので、状況をまとめてください
```

→ Cowork が状況サマリーを作る → コピーして CEO に LINE/メール

### 「直前の自分の変更を取り消したい」

```
自分が直前に行った変更を取り消してください。
先に何を取り消すかを見せてから、確認後に実行してください
```

## ✨ Cowork に話しかけるときのコツ

### コツ ①: 主語を「私は」「自分は」にする

```
× 「直して」
○ 「私は呪術廻戦の説明を直したいです。手伝ってください」
```

### コツ ②: 「先に見せて」を付ける (危険操作回避)

```
○ 「変更内容を先に見せてから、確認してから実行してください」
```

### コツ ③: 「分からないので教えて」と素直に言う

```
○ 「(用語) って何? 簡単に教えてください」
○ 「次に何をすればいいか分かりません。案内してください」
```

### コツ ④: 「壊れたかも」と思ったら手を止める

```
○ 「いったん止めてください。何が起きているか確認したいです」
```

### コツ ⑤: 「練習してみたい」と言う (本番反映回避)

```
○ 「練習で〇〇を試してみたいです。本番には反映しないでください」
```

## 📝 覚えなくていいこと

以下は **Cowork が裏で全部やってくれる** ので、覚えなくて OK:

- ファイルの場所 (例: `app/lib/astromeda-data.ts`)
- git の操作 (pull / commit / push 等)
- Shopify API の使い方
- TypeScript / React の文法
- Hydrogen / Oxygen の設定

→ あなたは **「何をしたいか」を日本語で伝えるだけ**。
