「コレクション」と「バナー」って何が違うの? なぜ分かれてるの?

最初によく出る質問です。1 つの図と 5 つの理由でスッキリ理解できます。

🏪 比喩で言うと: 「お店」=「倉庫」+「店頭」

📚
コレクション = 倉庫の棚
商品を分類して入れる箱・棚。常設で半永久的。
URL を持つ実体: /collections/kimetsu-no-yaiba
タグで自動振り分け。
🖼️
バナー = 店頭の看板・ポスター
「鬼滅の刃はこちら→」と案内する広告。
URL は持たず、クリックしたらどの棚に行くか(=リンク先)を指定。
季節やキャンペーンで貼り替える。
🛒 お客様の動き: トップページのバナー (看板) 👀 → クリック → コレクション (棚) で商品一覧 → 1 商品をクリック → 購入

📊 違いをひと目で比較

項目 📚 コレクション 🖼️ バナー
正体 商品を入れる箱 (データの実体) トップページの看板 (画像)
URL 自分で持つ /collections/... 持たない (リンク先を指定する側)
役目 商品をタグで自動的にまとめる お客様を「箱」へ呼び込む広告
必須? 必須 商品の住所 任意 なくても売れる
数 数百 (全 IP・全カテゴリ分) トップに見せたい厳選 8〜26 個
更新頻度 作ったら半永久的・自動更新 季節・キャンペーンで毎月差し替え
担当者 商品担当 (在庫・分類) マーケ担当 (デザイン・キャンペーン)
作る場所 (admin) 商品・販売 → 📚 コレクション コンテンツ・ページ → 📝 見た目 → IPコラボ / ヒーロー
関係 同じコレクションに複数バナーを作れる (例: 鬼滅コレクションに「春バナー」「夏バナー」)
1 つのバナーは 1 つのコレクションをリンク先に持つ

💡 なぜ分かれているか — 5 つの理由

  1. 役割が根本的に違う — コレクションは「実体・住所」、バナーは「看板・案内」。混ぜると変更が連鎖して壊れやすい。
  2. 作業の主導権が違う — コレクション = 商品担当 (タグ・分類) / バナー = マーケ担当 (デザイン・キャッチコピー)。担当者が違うので画面を分けた方が効率的。
  3. 更新頻度が違う — コレクションは作ったら放置・タグで自動更新。バナーは「春の鬼滅キャンペーン」「夏のセール」など毎月貼り替える。一緒の画面だと「うっかり棚を消した」事故が起きる。
  4. 1:N の関係 (再利用) — 同じコレクションに「春バナー」「夏バナー」「年末バナー」を作れる。コレクションを毎回作り直す必要なし。
  5. バナーなしでもお店は動く — でも「コレクションなし」は不可。商品の住所がないと検索もリンクもできない。だからコレクション = 必須・バナー = 任意で分離されている。
