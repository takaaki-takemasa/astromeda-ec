# 11. Cowork に頼めること・頼めないこと一覧

CEO や引き継ぎ先が「これは Cowork に頼んで終わるのか? 自分でボタンを押さないとダメなのか?」を 1 秒で判断できるよう、境界線を明確にした表です。

---

## 大原則

> **困ったら全部 Cowork に「これお願いできる?」と聞く**。Cowork は以下のいずれかで答えます:

| Cowork の答え | 意味 |
|---|---|
| 🟢 **「やります」** | Cowork が直接実行 (95% のケース) |
| 🟡 **「あなたの画面で 1 クリックお願いします、ここをこう押して」** | Cowork が画面を見ながら誘導・CEO は指示通り 1 クリックだけ |
| 🔴 **「これは私には絶対できません。あなたが [理由] で本人の手で実行が必須です」** | 本人のセキュリティ要素のみ・5% のケース |

CEO・メンバーは **「まず Cowork に聞く」** だけ覚えればよいです。

---

## 🟢 Cowork が完全に肩代わりできること (頼むだけで終わる)

### 商品・コレクション・コンテンツ

- 商品の追加・編集・削除
- 商品の値段変更・在庫変更
- 商品画像の差し替え
- コレクションの作成・編集・削除
- コレクションへの商品追加・除外
- IP コラボバナーの差し替え
- ヒーローバナーの追加・編集
- マーキーテキストの変更
- 記事 (ブログ・SEO) の追加・編集
- FAQ の追加・編集
- 法務情報・特商法・プライバシーポリシーの編集
- メニュー (ナビゲーション) の追加・編集
- URL リダイレクトの追加
- 一括タグ操作 (複数商品にタグ付与/除去)

### システム変更

- サイトのデザイン変更 (色・フォント・余白)
- 新機能の追加 (例: レビュー機能・Q&A 機能)
- バグ修正
- 画像の最適化・WebP 変換
- パフォーマンス改善
- SEO 対策 (メタタグ・構造化データ)

### デプロイ・運用

- 変更を本物のサイトに反映する (= デプロイ)
- 直前のバージョンに戻す (= ロールバック・`git revert`)
- GitHub Actions のログ確認
- エラーログの確認・原因調査

### Shopify 設定

- Shopify Hydrogen の環境変数表示 (Cowork が Chrome 経由で読む)
- 通常時の管理画面操作全般

### 管理画面 (admin) の操作

- 👥 メンバーの追加・編集・無効化・削除
- パスワード再設定
- 監査ログの閲覧

---

## 🟡 Cowork が誘導するが、CEO の 1 クリックが必要なこと

これらは「セキュリティ上、第三者 (= Cowork) が代行すべきでない」ものです。
Cowork が画面を Chrome MCP で開いて「ここを押してください」と指差し誘導します。

### CEO 自身のアカウント操作

- **GitHub branch protection 設定**
  - Cowork が GitHub Settings 画面を開く → CEO が「Save」ボタンを 1 度クリック
  - (理由: GitHub の設定変更は本人の判断ログを残す方が望ましい)

- **GitHub Collaborator 招待 (新メンバー追加)**
  - Cowork が Settings → Access 画面まで誘導 → CEO が「Add people」と「Invite」をクリック
  - (理由: 招待は CEO の意思表示として GitHub のログに残す)

- **OAuth 再認可** (新しい Shopify scope を追加した時)
  - Cowork が認可画面まで誘導 → CEO が「許可」ボタンを 1 度クリック
  - (理由: ストアオーナーの明示同意が必要)

- **Shopify Hydrogen 環境変数の変更** (例: `ADMIN_PASSWORD` を変更)
  - Cowork が Hydrogen Settings 画面を開く → CEO が新しい値を入力 → 「Save」クリック
  - (理由: 本番環境変数なので CEO が直接タイプする方が安全)

### CEO のスマホ・物理デバイスが必要なもの

- **GitHub 2FA の初期設定**
  - Cowork が手順を読み上げる → CEO がスマホの認証アプリで QR コードを読む
  - (理由: 2FA の本質はスマホという物理デバイス。Cowork はスマホを操作不可)

### Shopify サポートとのやり取り

- **Shopify サポートへの問い合わせ** (決済が動かない等)
  - Cowork が問い合わせ文を起案 → CEO がコピーして Shopify サポートチャットに貼り付け
  - (理由: Shopify アカウントの所有者本人とのやり取りが原則)

---

## 🔴 Cowork に頼めないこと (本人の手でしかできない)

これらは構造的に Cowork が実行不可能なものです (=「5%」)。

### 個人のセキュリティ要素

- **メンバー本人の GitHub アカウント作成**
  - 本人のメールアドレスとパスワードで本人がアカウント作成
- **メンバー本人の 2FA 設定**
  - 本人のスマホでないと QR コードが読めない
- **メンバー本人の GitHub PAT (アクセストークン) 発行**
  - 本人のアカウント設定画面でないと発行できない
- **メンバー本人による GitHub 招待 Accept**
  - 招待メールが本人に届いて本人が「Accept」をクリック

### Claude Desktop / Cowork のアカウント関連

- **Claude.ai Pro / Max への課金登録**
  - 本人のクレジットカードで登録
- **Claude Desktop アプリのインストール**
  - 本人のパソコンでアプリをダウンロードしてインストール

### 本人のパソコン操作

- **本人のパソコンで作業フォルダを選択** (Cowork に渡すフォルダ)
  - 本人がフォルダ選択ダイアログでクリック

---

## 🆘 「これはどっちに頼めばいいの?」と迷ったら

**5 秒判定**:

1. それは「サイトの中身を変える」ことか? → 🟢 Cowork に頼む
2. それは「GitHub・Shopify の設定を変える」ことか? → 🟡 Cowork に「やり方を教えて or 一緒にやって」
3. それは「自分のスマホ・メール・パスワード・カード」に関係するか? → 🔴 自分でやる (Cowork に手順だけ聞く)

---

## 引き継ぎ先メンバーが「私でも使える?」と不安な時

**答え**: はい、使えます。理由:

1. **覚える必要があるのは「Cowork に聞く」 だけ** — 5% の手動作業が来たら Cowork が画面を 1 ボタンずつ案内する
2. **間違っても直せる** — `09_困った時の解決ガイド.md` に症状別の対処を網羅
3. **CEO の引き継ぎ時はこのフォルダを丸ごと渡せばよい** — 引き継ぎ書を別途作る必要なし

このフォルダ全部 + Cowork (Claude Desktop) の組み合わせで、**新しい CEO は経歴ゼロで翌日から運用可能** です。

---

## メンバー (CEO 以外) の場合

メンバーは更にシンプル:

- 🟢 Cowork に頼める範囲 → ほぼ全部できる
- 🟡 Cowork が誘導が必要な範囲 → ほぼ来ない (CEO 用の設定だけ)
- 🔴 本人がやる範囲 → 加入時に GitHub アカウント作成・2FA・PAT 発行の 3 つだけ (`01_メンバー向け_セットアップ手順.md` の指示通り)

加入後の日常運用では、メンバーは **「Cowork に話しかける」 + 「管理画面でボタン押す」 だけ** です。
