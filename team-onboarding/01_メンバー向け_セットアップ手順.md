# Astromeda EC メンバー向け セットアップ手順 (v4)

ようこそ。このマニュアル通りに進めれば **約 30-40 分** で完了。
**途中で「こうしてください」と話しかけるだけで進む**ように設計されています。

**※ v4 (2026-04-24) 改訂**: A 演習で発見した「skill chicken-and-egg 問題」を修正しました。
clone 前でも Cowork が確実に動く一発テンプレに置き換え。GitHub Desktop は引き続き不要。

## 前提: 必要なもの

- 普段使う PC (Windows または Mac)
- メールアドレス
- 支払い用のカード (claude.ai Pro サブスク用)

## 全体の流れ (6 Step)

```
Step 1 — GitHub アカウント作成              (5分)
Step 2 — claude.ai Pro 登録                  (5分)
Step 3 — Claude Desktop インストール         (5分)
Step 4 — CEO の招待を承諾                    (5分)
Step 5 — 作業フォルダを作る                  (1分)
Step 6 — Cowork に自己紹介して開始           (15分・ここから話しかけるだけ)
```

## Step 1: GitHub アカウント作成 (5分)

ブラウザで `https://github.com/signup` を開く。

- メール入力 → Continue
- パスワード作成 → Continue
- **Username** を決める (英数字。例: `tanaka-astromeda`。後で変えられない)
- メールに届くコードを入力
- **"Free"** (無料プラン) を選ぶ

作成できたら、**GitHub の Username を CEO に LINE/メール** で送る。

> 例: 「GitHub username は `tanaka-astromeda` です」

## Step 2: claude.ai Pro 登録 (5分)

ブラウザで `https://claude.ai` を開く。

1. 右上 **Sign up** → メール登録
2. ログイン後、左下のあなたの名前 → **Settings** → **Plan**
3. **Pro プラン** (月 $20, 約 ¥3,000) を選んでカード決済
4. 領収書は CEO に提出 (経費精算)

## Step 3: Claude Desktop インストール (5分)

ブラウザで `https://claude.ai/download` を開き、あなたの OS (Windows/Mac) 用をダウンロード → インストール → 起動 → claude.ai と同じアカウントでログイン。

**Cowork mode を ON** にする:
- 左下のアバター → **Settings** → **Features**
- **Cowork mode** のトグルを ON
- 同意ダイアログに同意

## Step 4: CEO の招待を承諾 (5分)

CEO から **GitHub の招待メール** が届きます。件名:
> `@takaaki-takemasa invited you to collaborate on takaaki-takemasa/astromeda-ec`

1. メール内の **View invitation** ボタンを押す
2. ブラウザで **Accept invitation** を押す
3. 完了

## Step 5: 作業フォルダを作る (1分)

**Windows の場合**:
- エクスプローラーで `C:\Users\(あなたの名前)\Documents` を開く
- 右クリック → 新規 → フォルダー → 名前を **`astromeda-work`** にする

**Mac の場合**:
- Finder で「書類」を開く
- ファイル → 新規フォルダ → 名前を **`astromeda-work`** にする

## Step 6: Cowork に自己紹介して開始 (15分)

### 6-1. Claude Desktop で新しい会話を開く

Cowork mode が ON になっていることを確認 (左下の表示)。

さらに、左側のフォルダ選択欄で **Step 5 で作ったフォルダ** (`astromeda-work`) を選択。

### 6-2. 以下の文章をコピペして送る (v4 一発テンプレ)

> ⚠ **v4 重要修正**: 旧版では「自己紹介だけ送る」でしたが、Cowork が `.claude/skills/astromeda-team` という Astromeda 専用スキルを使うには、まずコードを clone する必要がある「鶏と卵」の問題がありました。
> v4 では、最初の自己紹介に **PAT 発行 → clone → スキル読み込み** の指示を全部入れます。これで素の Cowork でも確実に進めます。

```
はじめまして、Astromeda チームに参加した (あなたの名前) です。
GitHub の username は (あなたの GitHub username) です。

今からこのフォルダで Astromeda EC システムの運用作業を始めたいです。
以下を順番に手伝ってください:

1. まず私が GitHub から Personal Access Token (PAT) を発行する画面に
   案内してください。発行手順は「ブラウザで5クリック」レベルでお願いします。
2. 私が PAT を貼り付けたら、それを使って
   https://github.com/takaaki-takemasa/astromeda-ec.git
   このリポジトリを今のフォルダに clone してください。
3. clone が終わったら、リポジトリの中にある
   .claude/skills/astromeda-team/SKILL.md
   を読んで、以後はこのスキルに従って私を案内してください。
4. 練習として、自分の自己紹介を team-onboarding/members/ に追加して
   反映確認まで一緒にやりましょう。

何も知らない前提で、1 ステップずつ確認しながら進めてください。
```

このテンプレを送ると Cowork は:
- ① PAT 発行画面 (`https://github.com/settings/tokens/new?...`) を開く方法を案内
- ② 受け取った PAT を `~/.git-credentials` に安全に保存
- ③ git clone を実行 (約 30-60 秒)
- ④ clone した repo の中の **astromeda 専用 skill** を読み込んで、これ以降は IP コラボや商品の用語マッピング等、Astromeda 固有の作業も自動で扱えるようになります
- ⑤ 練習 push まで誘導

### 6-3. PAT 発行で詰まったとき

PAT 発行画面の入力例:
| 項目 | 値 |
|---|---|
| Note | `astromeda-ec-cowork` |
| Expiration | `90 days` (3ヶ月おきに再発行) |
| Scopes | ☑ `repo` (← これだけチェック) |

「Generate token」を押す → `ghp_xxxxxxx...` で始まる文字列が表示 → そのまま会話にコピペで OK。
**この文字列は 1 回しか表示されません**。失くしたら再発行になるので、Cowork に貼り付けるまで画面を閉じないでください。

### 6-4. 反映確認

練習 push 後、4-5 分待ってから:
- https://github.com/takaaki-takemasa/astromeda-ec/commits/main を開く
- 自分の名前で commit が表示されていれば成功 ✅

### 6-4. これ以降は、日本語で話しかけるだけ

```
「IP コラボの呪術廻戦の説明を直したい」
「トップのバナー文言を変えて」
「商品に新しいタグを付けたい」
```

Cowork が裏で全部やります。**git / コマンド / ファイルパスを覚える必要は一切ありません**。

## つまずいたときの質問フレーズ集

Cowork に以下のようにそのまま話しかけてください:

| 状況 | 言う内容 |
|---|---|
| 何をしていいか分からない | 「次に何をすればいいか分かりません」 |
| エラー画面が出た | 「(エラーメッセージ全文) と出ました」 |
| 操作に失敗した | 「失敗しました。どうすればいいですか?」 |
| 用語が分からない | 「(分からない単語) って何?」 |
| 壊したかもしれない | 「何か壊したかもしれません。止めてください」 |
| PAT って何? | 「PAT の発行方法を詳しく教えてください」 |

## それでも解決しないとき

**CEO (武正貴昭)** に LINE/メール で:
- いま何をしていて
- 何が起きたか
- 画面のスクショ

を送る → CEO が直接サポート or Cowork でリモート修復。
